## 2.- Escenario principal:

Escenario: Inicio de sesión exitoso

Actor: Usuario del sistema (Usuario solicitante, Product Manager o Desarrollador)

**Flujo principal**

1.  El usuario accede a la página principal del sistema.
2.  El sistema muestra la pantalla de inicio de sesión.
3.  El usuario ingresa su correo institucional.
4.  El usuario ingresa su contraseña.
5.  El usuario presiona el botón “Acceder al sistema”.
6.  El sistema valida las credenciales contra el mecanismo de autenticación institucional.
7.  Si las credenciales son válidas, el sistema permite el acceso.
8.  El usuario es redirigido a la pantalla principal del sistema (bandeja de solicitudes).

## 3\. Escenarios alternos

### Escenario alterno 1: Credenciales incorrectas

1.  El usuario ingresa correo o contraseña incorrectos.
2.  El sistema intenta validar las credenciales.
3.  El sistema detecta que la información es inválida.
4.  El sistema muestra un mensaje de error indicando que las credenciales no son correctas.
5.  El usuario permanece en la pantalla de inicio de sesión.

### Escenario alterno 2: Campos vacíos

1.  El usuario intenta acceder sin completar los campos obligatorios.
2.  El sistema detecta que falta información.
3.  El sistema muestra un mensaje solicitando completar los campos requeridos.

### Escenario alterno 3: Olvido de contraseña

1.  El usuario selecciona la opción “¿Olvidaste tu contraseña?”.
2.  El sistema redirige al mecanismo institucional de recuperación de contraseña.

## 2\. Escenarios

### Escenario 1 – Registrar una nueva solicitud

**Actor:** Usuario solicitante

**Flujo principal**

1.  El usuario accede al sistema e inicia sesión correctamente.
2.  El sistema muestra la pantalla Portal de Solicitudes.
3.  El usuario visualiza las opciones de tipo de solicitud disponibles.
4.  El usuario puede seleccionar alguno de los siguientes formularios:
    -   Reporte de Falla Urgente
    -   Modificación de Sistema
    -   Nuevo Sistema
5.  El usuario presiona el botón Iniciar Solicitud o Reportar Urgencia.
6.  El sistema redirige al formulario correspondiente para registrar la solicitud.

### Escenario 2 – Intentar acceder a Requerimientos Técnicos

**Flujo**

1.  El usuario visualiza la tarjeta **Requerimientos Técnicos**.
2.  El sistema indica que esta opción requiere aprobación previa.
3.  Si el usuario intenta acceder, el sistema mantiene el acceso bloqueado.
4.  El sistema muestra que el formulario estará disponible únicamente después de la aprobación del Product Manager.

**Resultado**

El usuario no puede acceder al formulario hasta que el proyecto haya sido autorizado.

### Escenario 3 – Consultar Mis Solicitudes

**Flujo**

1.  El usuario selecciona la opción **Mis Solicitudes** en el menú lateral.
2.  El sistema redirige al módulo donde se muestran todas las solicitudes registradas por el usuario.
3.  El usuario puede consultar el estado de sus solicitudes.

### Escenario 4 – Gestionar opciones de usuario

**Flujo**

1.  El usuario selecciona el menú de usuario ubicado en la parte superior derecha.
2.  El sistema despliega las opciones disponibles.
3.  El usuario puede seleccionar:
    -   Editar perfil (véase la pantalla **perfil de usuario**)
    -   Cerrar sesión
4.  El sistema ejecuta la acción seleccionada.

### Escenario 5 – Consulta de notificaciones

**Actor:** Usuario solicitante

**Flujo principal**

1.  El usuario se encuentra en la pantalla **Portal de Solicitudes**.
2.  El usuario observa el ícono de **notificaciones** en la barra superior.
3.  El usuario selecciona el ícono de notificaciones.
4.  El sistema despliega un panel o lista con las notificaciones recientes del usuario.
5.  El usuario puede revisar las notificaciones disponibles.
6.  Si el usuario selecciona una notificación, el sistema redirige a la sección correspondiente del sistema.

**Resultado**

El usuario puede consultar notificaciones relacionadas con sus solicitudes, como cambios de estado, aprobaciones o requerimientos adicionales.
